Separated Coding Project

Versi coding project yang sudah dipisah menjadi beberapa file agar lebih rapi, mudah dites, dan mudah dijelaskan saat exam.

ESP Ultrasonic MQTT Control ThingSpeak n8n Code n8n MQTT Processing Separated Code Credential Config Semua Coding WiFi Manager Kembali Home

Struktur File Terpisah

Gunakan struktur ini jika ingin memisahkan program utama menjadi modul WiFi, config, hardware, sensor, pompa, MQTT, ThingSpeak, dan main loop.

Lebih Rapi

Setiap file punya tugas sendiri, jadi lebih mudah dicari saat debugging.

Mudah Dites

Sensor, pompa, MQTT, dan ThingSpeak bisa dicek dari modul masing-masing.

Exam Friendly

Alur program lebih mudah dijelaskan karena setiap bagian punya nama file yang jelas.

Penjelasan Struktur Terpisah

config.py

Menyimpan credential, batas jarak air, mode relay, dan interval. File lain mengambil nilai dari sini supaya konfigurasi tidak tercecer.

hardware.py

Menyimpan deklarasi pin dan fungsi setup_pin(). Semua modul memakai pin dari file ini agar pin mapping konsisten.

sensor.py

Fokus pada DHT11 dan HC-SR04. Data sensor disimpan ke dictionary state agar bisa dipakai oleh pompa, MQTT, dan ThingSpeak.

pump.py

Fokus pada relay dan LED pompa. Fungsi set_pompa() dipakai untuk kontrol manual dari MQTT maupun kontrol otomatis dari sensor.

mqtt_service.py

Mengatur koneksi MQTT, callback command, publish data sensor, dan reconnect jika publish gagal.

thingspeak.py

Mengirim data dari state ke field ThingSpeak menggunakan HTTP GET.

main.py

Menjadi pusat alur program: konek WiFi, connect MQTT, baca sensor per interval, kontrol pompa, kirim MQTT, kirim ThingSpeak, dan cek command masuk.

Catatan Nama File

Nama file harus sama dengan import. Karena main.py memakai from mqtt_service import ..., file MQTT sebaiknya bernama mqtt_service.py.
