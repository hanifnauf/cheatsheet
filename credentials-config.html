Setting Credential Project

Gunakan halaman ini sebagai tempat referensi untuk mengganti broker MQTT, API key ThingSpeak, dan credential Telegram/n8n.

MQTT

Credential broker, port, client ID, publish topic, dan control topic.

ThingSpeak

Write API key, host API, dan mapping field untuk dashboard.

Telegram

Bot token dan chat ID untuk node Telegram di n8n atau script API.

1. MicroPython Config File: config.py

Catatan

Jika project dipublikasikan online, jangan tampilkan token Telegram atau API key asli. Gunakan placeholder seperti contoh di bawah.

# config.py
# Simpan file ini di ESP8266 bersama main.py dan wifimgr.py

# MQTT
MQTT_SERVER = "broker.emqx.io"
MQTT_PORT = 1883
MQTT_TOPIC = b"hanif/smarthydro"
MQTT_CONTROL_TOPIC = b"hanif/smarthydro/control"
CLIENT_ID = b"ESP8266-SmartHydro"

# Jika broker MQTT memakai username/password, isi bagian ini.
# Untuk broker public seperti broker.emqx.io, biasanya boleh None.
MQTT_USERNAME = None
MQTT_PASSWORD = None

# ThingSpeak
WRITE_API_KEY = "ISI_WRITE_API_KEY_THINGSPEAK"
THINGSPEAK_HOST = "api.thingspeak.com"

# Telegram
# Biasanya token dan chat ID dipakai di n8n, bukan di ESP8266.
TELEGRAM_BOT_TOKEN = "ISI_TOKEN_BOT_TELEGRAM"
TELEGRAM_CHAT_ID = "ISI_CHAT_ID_TELEGRAM"

Penjelasan Config

MQTT_SERVER

Alamat broker MQTT. Pada project ini broker yang dipakai adalah broker.emqx.io.

MQTT_TOPIC

Topic untuk publish data sensor dari ESP8266 ke n8n atau dashboard MQTT.

MQTT_CONTROL_TOPIC

Topic untuk menerima command dari Telegram/n8n. Topic ini harus sama dengan output node MQTT Publish di n8n.

WRITE_API_KEY

API key dari channel ThingSpeak. Key ini dipakai oleh fungsi kirim_thingspeak() untuk mengirim data ke field dashboard.

TELEGRAM_BOT_TOKEN

Token dari BotFather. Jika memakai node Telegram resmi di n8n, token ini biasanya disimpan di menu Credentials n8n, bukan ditulis langsung di Function node.

2. Cara Pakai config.py di main.py

import config

MQTT_SERVER = config.MQTT_SERVER
MQTT_PORT = config.MQTT_PORT
MQTT_TOPIC = config.MQTT_TOPIC
MQTT_CONTROL_TOPIC = config.MQTT_CONTROL_TOPIC
CLIENT_ID = config.CLIENT_ID

WRITE_API_KEY = config.WRITE_API_KEY
THINGSPEAK_HOST = config.THINGSPEAK_HOST

Penjelasan Penggunaan Config

import config

Memanggil file config.py. Setelah itu semua variabel credential bisa diambil dari file config.

Keuntungan

Jika broker MQTT atau API key berubah, cukup ubah config.py. Code utama tetap rapi dan lebih mudah dijelaskan saat exam.

3. MQTT dengan Username dan Password

from umqtt.simple import MQTTClient
import config

def connect_mqtt():
    client = MQTTClient(
        config.CLIENT_ID,
        config.MQTT_SERVER,
        port=config.MQTT_PORT,
        user=config.MQTT_USERNAME,
        password=config.MQTT_PASSWORD,
        keepalive=60
    )

    client.set_callback(mqtt_callback)
    client.connect()
    client.subscribe(config.MQTT_CONTROL_TOPIC)
    return client

Penjelasan MQTT Credential

user dan password

Digunakan jika broker MQTT private membutuhkan login. Jika broker public tidak membutuhkan login, nilai bisa dibuat None.

client.subscribe(config.MQTT_CONTROL_TOPIC)

ESP mendengarkan topic control dari config. Ini membuat topic lebih mudah diganti tanpa mencari manual di semua file.

4. ThingSpeak Field Configuration

# Mapping field ThingSpeak yang dipakai project
# field1 = suhu
# field2 = kelembaban
# field3 = level_air_persen
# field4 = status pompa, ON = 1 dan OFF = 0
# field5 = mode pompa, AUTO = 1 dan MANUAL = 0

path = (
    f"/update?api_key={WRITE_API_KEY}"
    f"&field1={suhu}"
    f"&field2={kelembaban}"
    f"&field3={level_air_persen}"
    f"&field4={pompa_value}"
    f"&field5={mode_value}"
)

Penjelasan ThingSpeak Credential

api_key

Parameter ini wajib ada di URL update. Tanpa API key yang benar, ThingSpeak tidak akan menerima data.

Mapping Field

Urutan field harus konsisten dengan dashboard. Jika field di ThingSpeak diubah, bagian mapping ini juga harus disesuaikan.

5. Telegram Credential di n8n

// Contoh config untuk Function node n8n jika ingin menyimpan referensi
// Token asli sebaiknya disimpan di menu Credentials n8n.

const telegramConfig = {
  botToken: "ISI_TOKEN_BOT_TELEGRAM",
  chatId: "ISI_CHAT_ID_TELEGRAM"
};

return [
  {
    json: {
      telegram_chat_id: telegramConfig.chatId,
      text: "Smart Hydroponic siap mengirim notifikasi."
    }
  }
];

Penjelasan Telegram Credential

botToken

Token bot didapat dari BotFather. Pada n8n, token ini lebih aman disimpan melalui Telegram Credentials, bukan ditulis langsung di code.

chatId

Chat ID menentukan ke akun atau grup Telegram mana pesan monitoring dikirim.

Workflow n8n

Alur umum: MQTT Trigger menerima data ESP, Code node memformat pesan, lalu Telegram node mengirim pesan menggunakan credential Telegram.
