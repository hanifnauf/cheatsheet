Parsing MQTT dan Status Emergency

Code ini cocok diletakkan pada node Code di n8n setelah MQTT Trigger.

Input

Mengambil topic dan message dari MQTT Trigger. Jika payload memakai nama lain seperti payload atau data, code tetap mencoba membacanya.

Proses

Payload diparse menjadi JSON, lalu nilai sensor diubah menjadi angka agar mudah divalidasi.

Output

Output berisi data rapi untuk Data Table, Google Sheet, atau Telegram, termasuk status Normal/EMERGENCY.

$input.first().json.message
$input.first().json.topic

// Ambil semua item dari MQTT Trigger
const items = $input.all();

const output = items.map(item => {
  const topic = item.json.topic || "";

  // Ambil payload dari MQTT Trigger
  let message =
    item.json.message ??
    item.json.payload ??
    item.json.data ??
    "";

  // Kalau payload masih Buffer/object aneh
  if (message && typeof message === "object" && message.data) {
    try {
      message = Buffer.from(message.data).toString();
    } catch (e) {
      message = "";
    }
  }

  // Parse JSON dari ESP8266
  let data = {};

  try {
    if (typeof message === "string") {
      data = JSON.parse(message.trim());
    } else if (typeof message === "object" && message !== null) {
      data = message;
    }
  } catch (error) {
    data = {};
  }

  // Ambil data sesuai coding ESP8266 terbaru
  const suhu = Number(data.temperature);
  const kelembaban = Number(data.humidity);
  const jarakAir = Number(data.distance_cm);
  const levelAir = Number(data.water_level_percent);

  const pumpStatus = data.pump_status || "UNKNOWN";
  const pumpMode = data.pump_mode || "UNKNOWN";

  // Validasi angka
  const suhuFinal = isNaN(suhu) ? null : suhu;
  const kelembabanFinal = isNaN(kelembaban) ? null : kelembaban;
  const jarakAirFinal = isNaN(jarakAir) ? null : jarakAir;
  const levelAirFinal = isNaN(levelAir) ? null : levelAir;

  // Waktu data masuk
  const tanggal = new Date().toISOString();

  // Deteksi kondisi darurat
  const emergencyMessages = [];

  // Cek level air berdasarkan persen
  if (levelAirFinal !== null) {
    if (levelAirFinal <= 5) {
      emergencyMessages.push("🚨 Air hampir habis!");
    } else if (levelAirFinal <= 20) {
      emergencyMessages.push("⚠️ Level air rendah!");
    } else if (levelAirFinal >= 95) {
      emergencyMessages.push("🌊 Tangki hampir penuh!");
    }
  }

  // Cek jarak ultrasonic
  // Jarak besar = air rendah
  // Jarak kecil = air penuh
  if (jarakAirFinal !== null) {
    if (jarakAirFinal >= 15) {
      emergencyMessages.push("⚠️ Jarak air tinggi, air rendah!");
    } else if (jarakAirFinal <= 8) {
      emergencyMessages.push("✅ Air cukup / penuh");
    }
  }

  // Cek suhu
  if (suhuFinal !== null) {
    if (suhuFinal >= 35) {
      emergencyMessages.push("🔥 Suhu terlalu tinggi!");
    } else if (suhuFinal <= 10) {
      emergencyMessages.push("❄️ Suhu terlalu rendah!");
    }
  }

  // Cek kelembaban
  if (kelembabanFinal !== null) {
    if (kelembabanFinal <= 20) {
      emergencyMessages.push("⚠️ Kelembaban terlalu rendah!");
    } else if (kelembabanFinal >= 90) {
      emergencyMessages.push("💧 Kelembaban terlalu tinggi!");
    }
  }

  // Status akhir
  let statusFinal = "Normal";

  if (emergencyMessages.length > 0) {
    const hanyaInfoAirCukup =
      emergencyMessages.length === 1 &&
      emergencyMessages[0].includes("Air cukup");

    statusFinal = hanyaInfoAirCukup ? "Normal" : "EMERGENCY";
  }

  // Output untuk Data Table / Google Sheet / Telegram
  return {
    json: {
      Tanggal: tanggal,
      Topic: topic,

      Suhu_C: suhuFinal,
      Kelembaban_persen: kelembabanFinal,

      Jarak_Air_cm: jarakAirFinal,
      Level_Air_persen: levelAirFinal,

      Status_Pompa: pumpStatus,
      Mode_Pompa: pumpMode,

      Status: statusFinal,
      Pesan_Emergency: emergencyMessages.length > 0
        ? emergencyMessages.join(" | ")
        : "-"
    }
  };
});

return output;

Penjelasan Detail Coding

$input.first().json.message dan $input.first().json.topic

Dua baris ini biasanya dipakai untuk mengecek isi data pertama dari MQTT Trigger. message adalah payload dari ESP8266, sedangkan topic adalah jalur MQTT tempat pesan diterima.

const items = $input.all();

Mengambil semua item yang masuk ke node Code. Ini penting karena MQTT Trigger bisa mengirim satu atau beberapa item, dan code akan memproses semuanya.

const output = items.map(item => { ... })

map() memproses setiap item MQTT satu per satu, lalu mengubahnya menjadi format output baru yang lebih rapi.

const topic = item.json.topic || "";

Mengambil topic dari data MQTT. Jika topic kosong atau tidak ada, digunakan string kosong agar code tidak error.

Pengambilan Payload

item.json.message ?? item.json.payload ?? item.json.data ?? "" berarti code mencoba membaca payload dari beberapa kemungkinan nama field. Operator ?? memilih nilai pertama yang bukan null atau undefined.

Konversi Buffer

Jika payload berbentuk object dengan properti data, code mencoba mengubahnya menjadi string memakai Buffer.from(message.data).toString(). Ini berguna ketika MQTT Trigger mengirim data dalam bentuk Buffer.

let data = {};

Membuat object kosong sebagai nilai awal. Jika parsing JSON gagal, data tetap object kosong sehingga bagian berikutnya tetap aman dijalankan.

Parse JSON

Jika message berupa string, code menjalankan JSON.parse(message.trim()). trim() menghapus spasi atau enter di awal dan akhir payload sebelum diparse.

Payload Sudah Object

Jika message sudah berbentuk object, code langsung memakai object tersebut sebagai data. Ini membuat code fleksibel untuk beberapa format input n8n.

catch (error) { data = {}; }

Jika payload bukan JSON valid, error ditangkap dan data dikosongkan. Tujuannya agar workflow n8n tidak berhenti hanya karena satu pesan rusak.

Ambil Data Sensor

Number(data.temperature), Number(data.humidity), Number(data.distance_cm), dan Number(data.water_level_percent) mengambil nilai dari payload ESP8266 lalu mengubahnya menjadi angka.

Status Pompa dan Mode Pompa

data.pump_status || "UNKNOWN" berarti jika status pompa tidak ada, tampilkan UNKNOWN. Hal yang sama dilakukan untuk pump_mode.

Validasi Angka

isNaN() mengecek apakah hasil konversi bukan angka. Jika bukan angka, nilai final dibuat null. Ini membuat output lebih jelas daripada menampilkan NaN.

const tanggal = new Date().toISOString();

Membuat timestamp saat data diproses. Format ISO cocok untuk database, Data Table, Google Sheet, dan proses filter waktu.

const emergencyMessages = [];

Membuat array kosong untuk menampung semua pesan kondisi darurat. Satu data bisa menghasilkan lebih dari satu pesan, misalnya air rendah dan suhu tinggi.

Cek Level Air Persen

Jika levelAirFinal tidak null, code mengecek tiga kondisi: air hampir habis saat kurang dari atau sama dengan 5 persen, air rendah saat kurang dari atau sama dengan 20 persen, dan tangki hampir penuh saat lebih dari atau sama dengan 95 persen.

Cek Jarak Ultrasonic

Jarak besar berarti permukaan air jauh dari sensor, sehingga air rendah. Jarak kecil berarti permukaan air dekat sensor, sehingga air cukup atau penuh.

Cek Suhu

Jika suhu lebih dari atau sama dengan 35 C, sistem memberi peringatan suhu tinggi. Jika suhu kurang dari atau sama dengan 10 C, sistem memberi peringatan suhu rendah.

Cek Kelembaban

Kelembaban kurang dari atau sama dengan 20 persen dianggap terlalu rendah. Kelembaban lebih dari atau sama dengan 90 persen dianggap terlalu tinggi.

let statusFinal = "Normal";

Status awal selalu Normal. Status baru berubah menjadi EMERGENCY jika ada pesan darurat yang benar-benar perlu diperhatikan.

hanyaInfoAirCukup

Bagian ini mengecek apakah satu-satunya pesan hanya "Air cukup". Jika iya, status tetap Normal karena itu bukan kondisi bahaya.

statusFinal = hanyaInfoAirCukup ? "Normal" : "EMERGENCY";

Ini adalah ternary operator. Jika hanyaInfoAirCukup benar, status tetap Normal. Jika salah, status berubah menjadi EMERGENCY.

Object Output

Bagian return { json: { ... } } membuat format output n8n. Semua field diberi nama yang jelas seperti Suhu_C, Level_Air_persen, dan Pesan_Emergency.

emergencyMessages.join(" | ")

Jika ada lebih dari satu pesan emergency, semua pesan digabung menjadi satu teks dengan pemisah garis vertikal. Jika tidak ada pesan, output diisi -.

return output;

Mengembalikan semua item hasil proses ke node berikutnya. Output ini bisa langsung disambungkan ke Google Sheets, Data Table, Telegram, atau node lain.
