Sensor, Level Air, dan Pompa

File ini dapat dijadikan inti main.py sebelum ditambahkan koneksi MQTT dan ThingSpeak.

Fungsi Utama

Membaca suhu, kelembaban, dan jarak air. Jarak dari sensor diubah menjadi persentase level air.

Logika Pompa

Jika mode AUTO dan jarak air melewati batas rendah, relay menyalakan pompa. Jika air sudah cukup, pompa dimatikan.

Catatan Exam

Pin ECHO HC-SR04 harus melewati pembagi tegangan karena ESP8266 hanya aman menerima input 3.3V.

Variabel Fungsi
TINGGI_WADAH_CM Tinggi wadah air sebagai dasar hitung persentase.
BATAS_JARAK_AIR_RENDAH Jarak besar berarti permukaan air rendah, sehingga pompa ON.
BATAS_JARAK_AIR_CUKUP Jarak kecil berarti permukaan air dekat sensor, sehingga pompa OFF. 
import time
import dht
import machine
from machine import Pin

DHT_PIN = Pin(14)
RELAY_PIN = Pin(5, Pin.OUT)
TRIG_PIN = Pin(16, Pin.OUT)
ECHO_PIN = Pin(4, Pin.IN)
LED_POMPA_OFF = Pin(13, Pin.OUT)
LED_POMPA_ON = Pin(15, Pin.OUT)

TINGGI_WADAH_CM = 20
BATAS_JARAK_AIR_RENDAH = 15
BATAS_JARAK_AIR_CUKUP = 8

RELAY_ON = 0
RELAY_OFF = 1

suhu = 0
kelembaban = 0
jarak_air_cm = 0
level_air_persen = 0
status_pompa = "OFF"
mode_pompa = "AUTO"

dht_sensor = dht.DHT11(DHT_PIN)

def setup_pin():
    RELAY_PIN.value(RELAY_OFF)
    TRIG_PIN.value(0)
    LED_POMPA_OFF.value(1)
    LED_POMPA_ON.value(0)

def update_led_pompa():
    if status_pompa == "ON":
        LED_POMPA_ON.value(1)
        LED_POMPA_OFF.value(0)
    else:
        LED_POMPA_ON.value(0)
        LED_POMPA_OFF.value(1)

def baca_jarak_once():
    try:
        TRIG_PIN.value(0)
        time.sleep_us(2)
        TRIG_PIN.value(1)
        time.sleep_us(10)
        TRIG_PIN.value(0)

        durasi = machine.time_pulse_us(ECHO_PIN, 1, 30000)
        if durasi < 0:
            return None

        return durasi / 58
    except Exception as e:
        print("Error ultrasonic:", e)
        return None

def baca_ultrasonic():
    total = 0
    jumlah = 0

    for i in range(3):
        jarak = baca_jarak_once()
        if jarak is not None and 1 < jarak < 400:
            total += jarak
            jumlah += 1
        time.sleep_ms(70)

    if jumlah == 0:
        return None

    return total / jumlah

def hitung_level_air(jarak):
    if jarak is None:
        return 0

    tinggi_air = TINGGI_WADAH_CM - jarak
    tinggi_air = max(0, min(tinggi_air, TINGGI_WADAH_CM))
    return int((tinggi_air * 100) / TINGGI_WADAH_CM)

def baca_sensor():
    global suhu, kelembaban, jarak_air_cm, level_air_persen

    try:
        dht_sensor.measure()
        suhu = dht_sensor.temperature()
        kelembaban = dht_sensor.humidity()
    except Exception as e:
        print("Gagal baca DHT:", e)

    jarak = baca_ultrasonic()
    if jarak is not None:
        jarak_air_cm = round(jarak, 1)
        level_air_persen = hitung_level_air(jarak_air_cm)
    else:
        jarak_air_cm = 0
        level_air_persen = 0

def kontrol_pompa_otomatis():
    global status_pompa

    if mode_pompa != "AUTO":
        return

    if jarak_air_cm == 0:
        print("Data ultrasonic tidak valid")
        return

    if jarak_air_cm > BATAS_JARAK_AIR_RENDAH:
        RELAY_PIN.value(RELAY_ON)
        status_pompa = "ON"
        update_led_pompa()
        print("Auto: air rendah, pompa ON")
    elif jarak_air_cm < BATAS_JARAK_AIR_CUKUP:
        RELAY_PIN.value(RELAY_OFF)
        status_pompa = "OFF"
        update_led_pompa()
        print("Auto: air cukup, pompa OFF")

setup_pin()

while True:
    baca_sensor()
    kontrol_pompa_otomatis()

    print("Suhu:", suhu, "C")
    print("Kelembaban:", kelembaban, "%")
    print("Jarak air:", jarak_air_cm, "cm")
    print("Level air:", level_air_persen, "%")
    print("Pompa:", status_pompa, "| Mode:", mode_pompa)

    time.sleep(2)

Penjelasan Detail Coding

import time, import dht, import machine

time dipakai untuk delay dan jeda pembacaan sensor. dht dipakai untuk membaca DHT11. machine dipakai untuk akses pin dan fungsi time_pulse_us() pada sensor ultrasonic.

Deklarasi Pin

DHT_PIN memakai GPIO14/D5, RELAY_PIN memakai GPIO5/D1, TRIG_PIN memakai GPIO16/D0, dan ECHO_PIN memakai GPIO4/D2. Pin relay dan TRIG dibuat output, sedangkan ECHO dibuat input.

Batas Ketinggian Air

TINGGI_WADAH_CM adalah tinggi wadah. BATAS_JARAK_AIR_RENDAH menentukan kapan pompa menyala. BATAS_JARAK_AIR_CUKUP menentukan kapan pompa mati. Pada ultrasonic, jarak makin besar berarti permukaan air makin rendah.

RELAY_ON = 0 dan RELAY_OFF = 1

Relay pada banyak modul ESP8266 bersifat aktif LOW. Artinya nilai 0 menyalakan relay dan nilai 1 mematikan relay. Bagian ini penting ditanya saat presentasi karena logikanya terbalik dari LED biasa.

setup_pin()

Fungsi awal untuk membuat sistem dalam keadaan aman: pompa mati, trigger ultrasonic LOW, LED OFF menyala, dan LED ON mati. Ini mencegah pompa langsung menyala saat ESP baru booting.

update_led_pompa()

Fungsi ini menyamakan LED indikator dengan status pompa. Jika status_pompa ON, LED ON menyala dan LED OFF mati. Jika status OFF, kondisi LED dibuat sebaliknya.

baca_jarak_once()

TRIG dibuat LOW sebentar, lalu HIGH selama 10 microsecond untuk memicu HC-SR04. Setelah itu ESP membaca lama pantulan pada ECHO. Rumus durasi / 58 mengubah waktu pantulan menjadi jarak dalam cm.

baca_ultrasonic()

Sensor dibaca 3 kali agar hasil lebih stabil. Nilai yang valid hanya jarak antara 1 sampai 400 cm. Jika semua pembacaan gagal, fungsi mengembalikan None.

hitung_level_air(jarak)

Jarak dari sensor dikurangi dari tinggi wadah untuk mendapatkan tinggi air. Nilai dibatasi agar tidak kurang dari 0 atau lebih dari tinggi wadah, lalu diubah menjadi persen.

baca_sensor()

Bagian ini membaca DHT11 untuk suhu dan kelembaban, lalu membaca ultrasonic untuk jarak air. Jika ultrasonic gagal, jarak dan level dibuat 0 supaya program tetap berjalan dan tidak crash.

kontrol_pompa_otomatis()

Fungsi hanya bekerja saat mode_pompa bernilai AUTO. Jika air rendah, relay ON. Jika air cukup, relay OFF. Jika air berada di tengah batas, status pompa dibiarkan agar relay tidak terlalu sering berubah.

while True

Loop utama membaca sensor, menjalankan kontrol pompa otomatis, mencetak data ke serial monitor, lalu menunggu 2 detik sebelum mengulang.
