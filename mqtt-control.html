Publish Data dan Subscribe Perintah

MQTT dipakai untuk komunikasi dua arah: ESP mengirim data sensor dan menerima perintah pompa dari n8n/Telegram.

Publish

ESP mengirim payload JSON ke hanif/smarthydro berisi suhu, kelembaban, jarak air, level air, status pompa, dan mode pompa.

Subscribe

ESP subscribe ke hanif/smarthydro/control untuk menerima command ON, OFF, AUTO, dan RESET_WIFI.

Loop MQTT

Versi terbaru mengecek pesan dengan mqtt_client.check_msg() langsung di loop utama agar command cepat diproses.

from umqtt.simple import MQTTClient
import machine
import time
import wifimgr

MQTT_SERVER = "broker.emqx.io"
MQTT_PORT = 1883
MQTT_TOPIC = b"hanif/smarthydro"
MQTT_CONTROL_TOPIC = b"hanif/smarthydro/control"
CLIENT_ID = b"ESP8266-SmartHydro"

mqtt_client = None

def mqtt_callback(topic, msg):
    global status_pompa, mode_pompa

    try:
        pesan = msg.decode().strip().upper()
    except Exception:
        print("Pesan MQTT tidak bisa dibaca")
        return

    print("\nPesan MQTT masuk")
    print("Topic:", topic)
    print("Pesan:", pesan)

    if topic != MQTT_CONTROL_TOPIC:
        return

    if "RESET_WIFI" in pesan:
        print("Reset WiFi...")
        wifimgr.delete_wifi_config()
        time.sleep(1)
        machine.reset()

    elif "AUTO" in pesan:
        mode_pompa = "AUTO"
        print("Mode pompa AUTO")

    elif "OFF" in pesan or "MATI" in pesan:
        mode_pompa = "MANUAL"
        status_pompa = "OFF"
        RELAY_PIN.value(RELAY_OFF)
        update_led_pompa()
        print("Pompa dimatikan manual")
        print("Relay value:", RELAY_PIN.value())

    elif "ON" in pesan or "NYALA" in pesan:
        mode_pompa = "MANUAL"
        status_pompa = "ON"
        RELAY_PIN.value(RELAY_ON)
        update_led_pompa()
        print("Pompa dinyalakan manual")
        print("Relay value:", RELAY_PIN.value())

    else:
        print("Perintah tidak dikenal")

def connect_mqtt():
    global mqtt_client

    try:
        print("\nMenghubungkan ke MQTT...")
        mqtt_client = MQTTClient(
            CLIENT_ID,
            MQTT_SERVER,
            port=MQTT_PORT,
            keepalive=60
        )

        mqtt_client.set_callback(mqtt_callback)
        mqtt_client.connect()
        mqtt_client.subscribe(MQTT_CONTROL_TOPIC)

        LED_MQTT.value(1)

        print("MQTT terhubung")
        print("Publish topic :", MQTT_TOPIC)
        print("Control topic :", MQTT_CONTROL_TOPIC)

        return mqtt_client

    except Exception as e:
        print("MQTT gagal:", e)
        LED_MQTT.value(0)
        return None

def kirim_mqtt():
    global mqtt_client

    if mqtt_client is None:
        mqtt_client = connect_mqtt()

    if mqtt_client is None:
        print("MQTT belum terhubung")
        return

    payload = (
        f'{{"temperature":{suhu},"humidity":{kelembaban},"distance_cm":{jarak_air_cm},'
        f'"water_level_percent":{level_air_persen},"pump_status":"{status_pompa}","pump_mode":"{mode_pompa}"}}'
    )

    try:
        mqtt_client.publish(MQTT_TOPIC, payload)
        LED_MQTT.value(1)

        print("\nMQTT terkirim:")
        print(payload)

    except Exception as e:
        print("MQTT publish gagal:", e)
        LED_MQTT.value(0)

        try:
            mqtt_client.disconnect()
        except Exception:
            pass

        mqtt_client = None

# Bagian ini berada di while True pada main.py
if mqtt_client is not None:
    try:
        mqtt_client.check_msg()
    except Exception as e:
        print("Gagal cek MQTT:", e)
        LED_MQTT.value(0)

        try:
            mqtt_client.disconnect()
        except Exception:
            pass

        mqtt_client = None

Penjelasan Detail Coding

MQTT_SERVER, MQTT_PORT, dan CLIENT_ID

Bagian ini menentukan broker MQTT yang dipakai, port koneksi, dan identitas ESP8266. CLIENT_ID harus unik agar broker tidak bingung membedakan perangkat.

MQTT_TOPIC dan MQTT_CONTROL_TOPIC

MQTT_TOPIC dipakai untuk mengirim data sensor. MQTT_CONTROL_TOPIC dipakai untuk menerima perintah dari n8n/Telegram.

msg.decode().strip().upper()

Pesan MQTT awalnya berupa byte. decode() mengubahnya menjadi teks, strip() menghapus spasi, dan upper() membuat command tetap terbaca walaupun diketik kecil atau besar.

Command RESET_WIFI

Jika pesan berisi RESET_WIFI, ESP menghapus konfigurasi WiFi melalui wifimgr.delete_wifi_config(), menunggu 1 detik, lalu restart dengan machine.reset().

Command AUTO

Mengubah mode_pompa menjadi AUTO sehingga pompa kembali dikendalikan oleh sensor ultrasonic dan batas air.

Command OFF dan ON

Kedua command ini mengubah mode menjadi MANUAL. OFF mematikan relay, ON menyalakan relay, lalu LED indikator pompa diperbarui dengan update_led_pompa().

connect_mqtt()

Fungsi ini membuat MQTT client, memasang callback, connect ke broker, subscribe ke topic control, menyalakan LED MQTT, dan mencetak topic publish/control ke serial monitor.

payload = (... f-string ...)

Payload terbaru memakai f-string agar data sensor masuk langsung ke format JSON. Field yang dikirim sama dengan yang dibaca n8n: temperature, humidity, distance_cm, water_level_percent, pump_status, dan pump_mode.

mqtt_client.publish(MQTT_TOPIC, payload)

Baris ini mengirim JSON ke broker. Jika berhasil LED MQTT menyala, jika gagal client diputus dan dibuat None supaya bisa connect ulang.

mqtt_client.check_msg()

Pada main terbaru, pengecekan pesan MQTT dilakukan langsung di loop utama. Ini membuat ESP sering mengecek command tanpa menunggu proses kirim data selesai.
